/**
36
 * This interface is used to access an abstract identifier for a process in the application layer.
37
 * It consists of multiple sub-ids each of these sub-ids is a pair of the form (fromChannel,
38
 * numChannels). The "fromChannel" represents from which channel of a tool place the record was
39
 * received. E.g. 0.2.1 (read right to left, in first layer received from channel 1, in second layer
40
 * received from channel 2, in third layer received from channel 0) So if a certain place receives a
41
 * channel-id and writes the channel-id from which it received the record to the id, it tells the
42
 * place which channels to use to ultimately arrive at the application process that created the
43
 * record. The second element in the pair "numChannels" is used to determine how many connections
44
 * are leading to the place where the record was received with "fromChannel". This piece of
45
 * information is crucial for reductions in order to determine whether they received information
46
 * from all partners that need to contribute to this reduction.
47
 *
48
 * This is encapsulated into this interface as the size of data needed for such an id may depend
49
 * on the actual system configuration (number of levels, number of channels per place, ...).
50
 *
51
 * Sorting is done on a per sub-id basis (using the fromChannel value) starting at the leftmost
52
 * sub-id (the one of highest order). E.g. 0.5 < 1.1; 0.2 < 0.3
53
 */
